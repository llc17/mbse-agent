# V2 优缺点文档

## V1 vs V2 对比

| | V1 | V2 |
|---|----|----|
| **架构** | 4 个函数顺序调用 | LangGraph 状态图 |
| **状态** | 无统一状态，靠函数参数传 | PipelineState 贯通全图 |
| **中断** | 不支持，一路到底 | 节点1/2 完成后可暂停确认 |
| **打回** | 不支持 | 带反馈回到对应节点重做 |
| **节点3 自修复** | while 循环，最多 2 次 | LangGraph 子图，最多 5 次 |
| **节点3 失败路由** | 只能往前走 | 可按失败原因回到节点1/2 |
| **Prompt 来源** | 手写，凭感觉 | 参考 SysML v2 + Modelica 官方库 |
| **可视化** | 无 | draw_mermaid() 出流程图 |
| **实验** | 手工跑几次 | 360 次自动批量 + 成功率曲线 |
| **温度** | 各节点硬编码 | 统一从 State 取，可参数化扫描 |
| **崩溃恢复** | 不支持 | LangGraph checkpoint 持久化 |

## V2 解决了什么

1. **人能参与** — 节点1、节点2 完成后暂停，用户确认/打回
2. **自修复更深** — 重试从 2 次提到 5 次，子图结构更清晰
3. **失败能溯源** — 仿真失败自动判断是需求问题还是 SysML 问题，路由回正确的节点
4. **代码更规范** — 三个阶段 prompt 都参考了官方库写法
5. **可实验** — 批量跑参数扫描，出论文图表
6. **可恢复** — 崩溃后从 checkpoint 继续，不用重头跑

## 已考虑到的问题（V2 内解决）

| 问题 | 措施 |
|------|------|
| 无限打回循环 | 加 `max_reject` 计数器，打回超过 3 次强制继续 |
| 实验用例非标准化 | 写 `test_cases.json`，固定标准输入文本 |
| 启动环境未检查 | 加 `check_prerequisites()`，缺 OMC/Python 库立刻报 |
| prompt 版本未记录 | 每次运行把 prompt 模板复制到 run_dir |
| LLM 调用无日志 | logging 模块记录每次 API 调用的 token 数和时间 |
| 实验/交互模式切换 | 命令行参数 `--mode interactive/experiment`，实验模式跳过 HITL |
| 实验中途崩溃 | 实验脚本记录已完成的 trial，支持 `--resume` |
| 成功率定义模糊 | 统一：节点3 compile+simulate 都通过 = 成功 |
| 节点1 实验模式 | 跳过 input() 对话，直接用完整文本一次出 req |
| prompt 占位符太死 | 改为通用注入：`{req_json}` 直接传整个 req 对象 |
| API 调用无重试 | llm_client 加 retry + exponential backoff |
| 磁盘空间 | 实验模式只保留 result.json，不存 checkpoint |
| State 序列化 | Pydantic ↔ dict 转换，checkpoint 前 dump，恢复后 validate |
| LangGraph 版本 | lock 版本号确保可复现 |
| 代码重复 | `_load_prompt`、`_clean_code_block` 等提取到 `src/utils.py` |
| 实验耗时太长 | 先跑小规模 4×3×5=60 次验证，再加 `--parallel` 并发 |
| 无耗时统计 | PipelineState 加 `timing` 字段，记录每节点耗时 |

## V2 的局限（已知但不在这版解决）

| 局限 | 影响 | 何时解决 |
|------|------|---------|
| 只覆盖 2 个物理域（电气+热） | 液压/电机/流体等无法建模 | V3 加 3-5 用例，V7 RAG |
| 没有物理正确性验证 | 仿真通了但结果可能算错 | V6 LLM-as-Judge |
| 没有语法严格校验（ANTLR） | 语法检查只靠正则，漏检率高 | V7 |
| 没有知识库动态检索 | 示例硬编码，官方标准更新后过时 | V7 RAG |
| 命令行交互 | 体验差，看 JSON 费劲 | V4 Streamlit |
| 评估指标缺失 | 只有"成功率"，没有物理精度等维度 | V6 |
| DeepSeek 单点依赖 | 限流则全挂 | V4 加备用模型 |
| 自修复可能倒退 | 修好编译又坏了仿真，无法检测 | V7 加修复质量检查 |
| 编译通过但方向错了 | LLM 为过编译改物理方程，RC 低通修成高通 | V6 仿真值 vs 理论值校验 |
| SysML v2 语法正确率未知 | 若 <30%，需触发 Plan B | Day 23-24 决策点 |
| 对比不同模型不方便 | 想对比 deepseek-chat vs reasoner 需要改代码 | V6 |
